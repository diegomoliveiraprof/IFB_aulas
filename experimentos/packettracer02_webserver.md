# Experimento Packet Tracer - cenário com webserver

## OBJETIVO
O objetivo desta atividade é proporcionar ao estudante uma **experiência prática mais completa** com o Cisco Packet Tracer, indo além de uma rede local simples.  

Com este experimento, o aluno irá:

- **Explorar cenários pré-configurados**, entendendo como diferentes redes podem se interligar.  
- **Adicionar e configurar dispositivos finais**, reforçando o conceito de endereçamento IP.  
- **Testar conectividade interna e externa**, compreendendo a diferença entre comunicação local e acesso a recursos externos.  
- **Experimentar a configuração de servidores** dentro do Packet Tracer, como Web e DNS.  
- **Simular o acesso a serviços de Internet**, visualizando como ocorre a resolução de nomes e a navegação em páginas web.

1. Abrir o arquivo `.pkt` fornecido, que contém um cenário pré-configurado com duas redes:  
   - Uma **rede local**.  
   - Uma **rede simulando a Internet**, com um **servidor Web** e um **servidor DNS**.  
2. Adicionar **5 dispositivos finais**.  
3. Configurar o **endereçamento IPv4**.  
4. Testar a **conexão interna** utilizando o comando `ping`.  
5. Verificar a possibilidade de **configuração de um servidor** no Packet Tracer.  
6. Testar a **conexão externa** com `ping` (via endereço IP e via URL).  
7. Testar o **acesso ao servidor Web** via navegador.
8. Observar e analisar os pacotes.
